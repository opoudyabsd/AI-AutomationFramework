# TC-API-CORE-002: Confirm setState overwrites graph settings not constructor options

**Summary:** Verifies documented persistence semantics where graph settings are replaced by `setState()` while constructor-level configuration options persist.

**Priority:** 5 - Incorrect assumptions about persistence can create subtle regressions in state restore flows.

**Source:** docs/desmosAPI/Desmos-api-v1.11.md - Configuration Options vs Graph Settings; GraphingCalculator.getState(); GraphingCalculator.setState(obj, [options])

**Related:** TC-API-CORE-001

## Preconditions

- GraphingCalculator instance is initialized with constructor option `{ expressions: false }`.
- A baseline graph setting is set via `updateSettings({ degreeMode: true })`.
- A serializable state object captured from another calculator context has `degreeMode: false`.

## Test Steps

| Step | Action | Expected Result |
|------|--------|-----------------|
| 1 | Initialize calculator with `{ expressions: false }`. | Expressions list UI remains disabled per constructor option. |
| 2 | Update setting to `degreeMode: true` and verify `calculator.settings.degreeMode`. | Setting reflects `true`. |
| 3 | Call `calculator.setState(savedStateWithDegreeModeFalse)`. | State is applied successfully. |
| 4 | Read `calculator.settings.degreeMode` and observe expressions-list visibility behavior. | `degreeMode` becomes `false` (state overwritten), while constructor behavior for `expressions: false` remains in effect. |

## Notes for Automation

- **API surface:** `updateSettings`, `settings`, `getState`, `setState`.
- **Setup strategy:** Build two calculator instances in the same page: one to generate saved state payload, one to validate load behavior.
- **Assertion strategy:** Assert value transitions on observable settings plus stable UI behavior corresponding to constructor option.
- **Data considerations:** Use explicit state payload generated by API, not hand-authored JSON, because docs treat state as opaque.
- **Cleanup:** Destroy both calculator instances at end of test.
